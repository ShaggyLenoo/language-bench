Language Bench

A systematic performance comparison of C++ and Rust

Overview Benchmarks Methodology Results Summary GitHub

Overview

This project presents a detailed performance benchmarking study comparing C++ and Rust across workloads commonly found in systems programming.

Instead of relying on synthetic microbenchmarks from the internet, this study builds a reproducible benchmarking framework that evaluates execution behavior under controlled conditions, with multiple runs and statistical analysis.

Benchmarks Implemented

  • Summation Loop
  • Branch-Heavy Loop
  • Fibonacci Recursion
  • File Read
  • File Write
  • String Parsing
  • String Concatenation & Substring Search
  • Dynamic Allocation and Free
  • Vector Insertion and Deletion

Methodology

  • Identical algorithms and workloads for both languages
  • Optimized compilation (-O3 for C++, -O for Rust)
  • Each benchmark executed multiple times
  • Average execution time and standard deviation recorded
  • Large workloads executed using chunking to respect memory limits

Benchmark Results and Analysis

Each chart compares average execution time for C++ and Rust. Lower values indicate better performance.

1. Summation Loop

This benchmark measures raw arithmetic throughput and loop execution efficiency. It highlights how well each compiler optimizes tight, CPU-bound loops.

2. Branch-Heavy Loop

This benchmark evaluates performance under frequent conditional branching. Differences here are primarily influenced by branch prediction and control-flow handling.

3. Fibonacci Recursion

Recursive workloads stress function call overhead and stack management. This benchmark exposes differences in recursion handling and call optimization.

4. File Read

Measures sequential disk read throughput. Since this is I/O-bound, results reflect buffering strategies and system call overhead.

5. File Write

Evaluates sequential file write performance. OS-level buffering and flush behavior strongly influence these results.

6. String Parsing

This benchmark tests character-by-character parsing over large strings, representing text-heavy workloads such as log processing.

7. String Concatenation and Substring Search

Measures the combined cost of growing strings and searching within them, highlighting allocation behavior and string implementation efficiency.

8. Dynamic Allocation and Free

Focuses purely on heap allocation and deallocation performance, exposing allocator efficiency under heavy memory churn.

9. Vector Insertion and Deletion

Tests dynamic container manipulation under large workloads, stressing memory growth, element movement, and cache behavior.

Results Summary

  • C++ performs well in tight arithmetic and some recursive workloads
  • Rust excels in branch-heavy, string-processing, and memory-intensive benchmarks
  • Performance differences grow with workload size
  • No language is universally faster — workload characteristics dominate

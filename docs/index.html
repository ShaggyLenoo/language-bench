Language Bench

A systematic performance comparison of C++ and Rust across CPU, I/O, string processing, and memory workloads.

Overview Methodology Results System Repository

Overview

This project evaluates the runtime performance characteristics of C++ and Rust using a consistent, reproducible benchmarking framework.

Rather than relying on external benchmark claims, all measurements were produced locally using equivalent implementations, repeated runs, and statistical aggregation.

Benchmarking Methodology

  • Identical algorithms and workloads in both languages
  • Optimized compilation (-O3 for C++, -O for Rust)
  • Multiple executions per benchmark
  • Average execution time and standard deviation recorded
  • Large workloads executed using chunked processing

Benchmark Results

All charts below show average execution time. Lower values indicate better performance.

1. Summation Loop

Measures raw arithmetic throughput and loop execution efficiency in a tight, CPU-bound workload.

2. Branch-Heavy Loop

Evaluates performance under frequent conditional branching, highlighting branch prediction and control-flow efficiency.

3. Fibonacci Recursion

Measures recursive function call overhead and stack behavior under varying recursion depths.

4. File Read

Evaluates sequential file read throughput for large files, primarily influenced by OS buffering and I/O behavior.

5. File Write

Measures sequential file write performance and runtime I/O overhead across different file sizes.

6. String Parsing

Evaluates character-by-character string parsing performance, representative of real-world text processing workloads.

7. String Concatenation and Substring Search

Measures the combined cost of repeated string concatenation and substring search operations.

8. Dynamic Allocation and Free

Evaluates heap allocation and deallocation performance under increasing allocation pressure.

9. Vector Insertion and Deletion

Measures dynamic container behavior under repeated insertions and deletions, stressing memory growth and element movement.

System Configuration
